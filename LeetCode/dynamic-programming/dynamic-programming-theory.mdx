## 什么是动态规划

动态规划是一种把大问题拆成小问题，再利用小问题结果求解大问题的方法。

它的核心不是暴力枚举所有可能，而是识别重复子问题，并把已经计算过的结果保存下来，避免重复计算。

## 动态规划的关键思想

动态规划通常包含两个核心点：

- 最优子结构
- 重复子问题

如果一个问题的最优解可以由子问题的最优解推出来，并且子问题会重复出现，那么它就很适合用动态规划处理。

## 动态规划为什么难

动态规划的难点通常不在代码，而在建模。

你需要想清楚这些问题：

- `dp[i]` 或 `dp[i][j]` 表示什么
- 状态转移方程是什么
- 初始化条件是什么
- 遍历顺序为什么这样安排

一旦这些点想清楚，代码往往只是表达过程。

## 动态规划和递归的关系

很多动态规划问题都可以先写成递归，再进一步优化成记忆化搜索或自底向上的 DP。

所以动态规划并不是和递归完全对立，它更像是在递归思路上加入缓存与状态设计，让计算过程更高效。

## 常见问题类型

动态规划题常见可以分成几类：

- 线性 DP
- 背包问题
- 子序列问题
- 区间 DP
- 树形 DP
- 状态压缩 DP

初学阶段最重要的是先把线性 DP 和背包问题打牢，因为很多题的思维方式都从这里开始。

## 状态转移方程是什么

状态转移方程就是“当前状态如何由之前状态推出来”。

例如你定义了 `dp[i]` 表示到第 `i` 个位置时的最优解，那么你就必须回答：

- `dp[i]` 从哪些状态转移而来
- 为什么这些状态足够
- 为什么这样转移不会漏掉最优解

这一步是动态规划的核心。

## 初始化为什么重要

很多动态规划题不是思路错，而是初始化错。

如果初始状态定义不清楚，后面的转移就会全部偏掉。所以做 DP 时，通常要先单独思考：

- `dp[0]` 是什么
- 空数组、空字符串时结果是什么
- 第一行第一列应该怎么填

## 动态规划和贪心的区别

动态规划更适合处理“当前选择会影响未来很多状态”的问题。

贪心则更强调“当前这一步直接选最优”。  
如果题目无法证明局部最优可以推出全局最优，那往往就要考虑动态规划。

## 学习建议

如果你刚开始学动态规划，建议按这个顺序练习：

- 爬楼梯
- 打家劫舍
- 不同路径
- 01 背包
- 最长递增子序列
- 最长公共子序列

当你开始习惯先问“状态是什么、转移是什么”时，动态规划就会慢慢从抽象变得具体。
