## 什么是链表

链表是一种线性数据结构。

和数组不同，链表中的元素不需要连续存放。每个节点除了保存自己的值，还会保存指向下一个节点的引用。也正是因为这个特点，链表在插入和删除时通常更灵活，但在随机访问时不如数组。

## 常见类型

常见的链表主要有四种：

- 单链表
- 双链表
- 循环链表
- 双向循环链表

单链表的节点只知道下一个节点是谁。双链表的节点同时知道前一个和后一个节点。循环链表的尾节点会重新指向头节点，适合处理环形问题或循环访问场景。

## 存储方式

数组在内存中通常是连续存储的，因此你可以通过下标快速定位一个元素。

链表不是这样。链表节点在内存中通常是分散的，节点之间依靠指针或引用连接起来。所以链表查找某个位置的元素时，往往需要从头开始一步一步向后走。

这也是链表和数组最关键的差别：

- 数组适合查询
- 链表适合插入和删除

## 节点定义

链表题目的第一步通常是理解节点长什么样。

一个最基础的单链表节点通常包含两部分：

- 当前节点的值
- 指向下一个节点的指针

以常见的伪代码为例：

```text
Node {
  value
  next
}
```

如果是双链表，还会多一个 `prev` 指针，用来指向前一个节点。

## 常见操作

### 查询

链表不支持像数组那样通过下标直接访问第 `k` 个元素。

如果你想找到某个节点，通常只能从头节点开始逐个遍历，所以查询第 `k` 个节点的时间复杂度一般是 `O(n)`。

### 插入

链表的插入关键不在于移动元素，而在于修改指针指向。

如果你已经拿到了待插入位置前一个节点，那么插入一个新节点通常只需要常数次指针调整。

### 删除

删除操作的本质也是修改指针。

如果你知道待删除节点的前一个节点，就可以把前一个节点直接连到下一个节点上。问题的重点通常不是“删”，而是“怎么找到前一个节点”。

## 为什么链表题常用虚拟头节点

很多链表题会先引入一个虚拟头节点，也叫 dummy node。

这样做的好处是可以统一处理头节点被修改的情况。尤其是在删除头节点、在头部插入节点时，代码会简单很多。

## 时间复杂度对比

链表和数组的常见复杂度对比如下：

- 随机访问：数组更快，链表更慢
- 头部插入删除：链表更合适
- 中间插入删除：如果已知目标位置前驱节点，链表更方便

所以并不是链表一定比数组好，而是它们适合不同问题。

## 学习建议

如果你刚开始刷链表题，建议先把这几类题练熟：

- 删除节点
- 反转链表
- 两两交换节点
- 快慢指针
- 环形链表

当你真正理解了“链表的本质是改指针”，后面的很多题都会变得清晰。
