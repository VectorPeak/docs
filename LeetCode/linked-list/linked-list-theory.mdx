## 核心认知

链表是一种线性数据结构，但它和数组最大的不同在于：

- 数组中的元素通常连续存储
- 链表中的节点通常分散存储
- 链表依靠指针或引用把节点串起来

所以链表的核心不是“按下标访问”，而是“改指针”。

## 为什么要学链表

链表是很多基础题型的起点。它会逼你真正理解：

- 指针移动
- 局部修改
- 虚拟头节点
- 快慢指针

如果这些点没有吃透，后面的树、图、回溯很多时候也会写得不稳定。

## 节点长什么样

最基础的单链表节点只需要两个字段：

- 当前节点的值
- 指向下一个节点的引用

```python
class ListNode:
    def __init__(self, val: int = 0, next: "ListNode | None" = None):
        self.val = val
        self.next = next
```

如果是双链表，通常还会多一个 `prev` 指针。

## 链表和数组的区别

这两者最值得记住的区别只有一条：

- 数组擅长查
- 链表擅长改

更具体一点：

- 数组随机访问通常是 `O(1)`
- 链表随机访问通常是 `O(n)`
- 链表在已知前驱节点时，插入和删除通常是 `O(1)`

所以很多链表题看起来在“删除节点”，本质其实是在问你“能不能稳定地找到前驱节点”。

## 链表题为什么总喜欢用 dummy node

很多链表题的第一步，就是先创建一个虚拟头节点。

这样做的好处是：头节点不再是特殊情况。删除头节点、头插法、区间反转时，代码会更统一，也更不容易错。

```python
class ListNode:
    def __init__(self, val: int = 0, next: "ListNode | None" = None):
        self.val = val
        self.next = next


def remove_elements(head: ListNode | None, target: int) -> ListNode | None:
    dummy = ListNode(0, head)
    prev = dummy
    cur = head

    while cur:
        if cur.val == target:
            prev.next = cur.next
        else:
            prev = cur
        cur = cur.next

    return dummy.next
```

这段代码的关键不是删除本身，而是始终维护好 `prev` 和 `cur` 的关系。

## 链表常见题型

链表题经常围绕这些模型展开：

- 删除节点
- 反转链表
- 两两交换节点
- 快慢指针
- 环形链表
- 链表相交

虽然题目很多，但共性通常很强。你每次都可以先问自己：

1. 现在要操作哪个节点？
2. 我有没有前驱节点？
3. 这里需不需要 dummy node？

## 一个最经典的 Python 例子

反转链表几乎是所有链表题的基础。

```python
class ListNode:
    def __init__(self, val: int = 0, next: "ListNode | None" = None):
        self.val = val
        self.next = next


def reverse_list(head: ListNode | None) -> ListNode | None:
    prev = None
    cur = head

    while cur:
        nxt = cur.next
        cur.next = prev
        prev = cur
        cur = nxt

    return prev
```

这段代码就是链表思维最纯粹的体现：保存下一跳、反转指针、整体后移。

## 学习建议

如果你刚开始刷链表题，建议按这个顺序来：

- 删除元素
- 反转链表
- 两两交换节点
- 删除倒数第 N 个节点
- 环形链表

当你开始习惯从“指针怎么连”而不是“值怎么变”去思考时，链表题就真正入门了。
