## 什么是二叉树

二叉树是一种树形数据结构。

它的特点是每个节点最多只有两个子节点，通常称为左孩子和右孩子。很多算法题中的递归、分治和遍历思想，都会以二叉树作为入门模型。

## 二叉树的基本结构

一个二叉树节点通常包含三部分：

- 当前节点的值
- 左子节点指针
- 右子节点指针

最基础的结构可以理解为：

```text
TreeNode {
  value
  left
  right
}
```

理解这个结构之后，很多题目其实都可以归结为“如何处理当前节点，以及如何处理左右子树”。

## 二叉树的特点

二叉树和数组、链表最大的不同在于它不是线性结构，而是层级结构。

这种结构非常适合表达：

- 递归关系
- 分治问题
- 层级遍历
- 路径问题

所以二叉树不仅本身重要，也经常是学习递归思想最好的练习对象。

## 常见遍历方式

二叉树最基础的内容就是遍历。

深度优先遍历主要有三种：

- 前序遍历
- 中序遍历
- 后序遍历

除此之外，还有：

- 层序遍历

前中后序的区别，本质上就是“访问当前节点”的时机不同。层序遍历则通常借助队列，一层一层向下处理。

## 为什么二叉树题适合递归

二叉树天然具有“子问题结构相同”的特点。

如果你会处理一棵树，那么你通常也会处理它的左子树和右子树。所以很多二叉树题都可以先写出递归函数，再思考：

- 递归函数的参数是什么
- 返回值是什么
- 单层逻辑是什么

这也是很多人第一次真正理解递归的地方。

## 常见问题类型

二叉树题常见可以分成几类：

- 遍历类
- 深度和高度类
- 路径类
- 构造类
- 二叉搜索树类
- 最近公共祖先类

虽然题目很多，但本质上通常都绕不开遍历和递归。

## 二叉树和二叉搜索树的区别

二叉树不要求节点值满足大小关系。

二叉搜索树则额外要求：

- 左子树所有节点值小于根节点
- 右子树所有节点值大于根节点

这个性质决定了二叉搜索树在查找、插入和删除时有更强的结构优势。

## 学习建议

如果你刚开始刷二叉树题，建议先按下面顺序学习：

- 前中后序遍历
- 层序遍历
- 最大深度和最小深度
- 翻转二叉树
- 路径总和
- 二叉搜索树基础题

当你真正理解了“二叉树题的核心是明确递归函数定义”，很多题会自然串起来。
